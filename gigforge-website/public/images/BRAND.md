## Logo

### Files

| File | Usage |
|------|-------|
| `public/images/logo.svg` | Full logo (logomark + logotype) — header desktop, proposals, email signatures |
| `public/images/logo-icon.svg` | Icon-only mark — header mobile, favicons, app icons, small contexts |
| `public/favicon.svg` | Browser tab favicon (32×32) |

### Logomark Design

A hexagonal shape in electric blue (`#3b82f6`) with a white bold "G" letterform. The hexagon conveys technical precision, structure, and engineering rigour. The "G" is recognisable at small sizes and works as a standalone mark.

### Logotype

"GigForge" in system sans-serif bold, text-primary (`#f8fafc`), adjacent to the logomark.

### Logo Usage Rules

**Do:**
- Use full logo on white/light backgrounds with sufficient padding (min 16px clear space on all sides)
- Use full logo on dark backgrounds — this is the primary usage context
- Use icon-only mark when space is constrained (< 120px wide)
- Maintain aspect ratio at all times

**Don't:**
- Change the logo colours (no blue text, no dark hex)
- Stretch, rotate, or distort the logo
- Place logo on busy backgrounds without a background container
- Use logo below 24px height (icon) or 80px width (full logo)

|---------|------|--------|
| Service cards | 32×32 | 1.5px, electric blue |
| Navigation / UI | 20×20 | 1.5px, text-secondary |
| Feature badges | 24×24 | 1.5px, electric blue |

---

## Image Style

All images on the GigForge website must:
1. Work on dark navy (`#0f172a`) backgrounds — no white-bg photos
2. Use a cool colour temperature (blues, greys, dark tones)
3. Convey technical excellence — avoid generic stock-photo clichés
4. Be royalty-free (Unsplash, Pexels, or hand-crafted SVG)

---

## Voice

- **Confident.** We don't hedge. "We deliver X" not "We try to deliver X."
- **Technical but accessible.** Use precise language, define jargon.
- **Proof-first.** Lead with evidence (test counts, delivery times, client results).
